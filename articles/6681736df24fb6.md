Tauri を使って Windows 用のマウスジェスチャーアプリを作ったので、それについて紹介する。

特に、Tauri を薄く使うことで、常駐時のメモリ使用量を数 MB まで抑えたので、これについて説明する。

## 技術スタック

本ソフトは、Tauri をベースとしている。

Tauri とは、雑に言うと Rust 版 Electron であり、以下の特徴がある。

|  | Electron | Tauri |
| -- | -- | -- |
| ランタイム | Node.js | Rust |
| 画面描画 | Chromium | WebView |

Tauri は、Node.js や Chromium といったヘビーなものを内蔵しない。これにより軽量さを実現している。
しかし、OS間の差分の吸収などは Electron に比べると難しく、決して銀の弾丸という訳ではない。

### ベース（ジェスチャー認識・軌跡描画）

ここは、ジェスチャーの認識や、マウスの軌跡を描画するコアな部分となる。

この部分は全て Rust で実装している。
奇跡の描画には Windows API を直接叩いている。
ここが、本ソフトの常駐時のメモリ使用量が極めて低い理由である。

常駐する部分ではJSを一切使わず、WebViewを開く必要がないようにすることで、Rustのネイティブアプリと同等のパフォーマンスを実現している。

Tauriは、必ずしもWebViewを開く必要があるわけではないので、このようにRustから直接Windows APIを使って画面を描画するといった芸当もできるのだ。

### 設定画面

設定画面は、WebViewを使って実装している。

設定画面は、常に開くわけではないので、パフォーマンスより開発の容易さを優先した。

設定画面を開いている間は、メモリ使用量が100MB近くまで上がるが、閉じれば数MBまで戻る。
